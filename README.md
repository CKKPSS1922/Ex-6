# ⚔ Врата Судьбы

Онлайн-доска для игры в **Dungeons & Dragons**. Мастер ведёт карту, игроки двигают токены, бросают кубы, слушают музыку — всё в реальном времени через WebSocket.

## ✨ Возможности

- 🗡️ **Игроки** — создают героя (имя, цвет токена, аватар), свободно перемещают токен по карте (без привязки к клеткам)
- 👑 **Мастер** — пароль `Ничтожество`, полный контроль над миром:
  - импорт карты (любое изображение)
  - настраиваемая сетка
  - рисование, ластик, фигуры (прямоугольник, круг, линия)
  - создание и управление NPC
  - **индивидуальный туман войны для каждого игрока** — можно скрывать/открывать области для конкретных персонажей
  - загрузка и воспроизведение музыки — играет одновременно у всех
- 💬 **Чат** с бросками кубов (`/r 1d20+5`) и приватными сообщениями мастеру (`/w текст`)
- 🌫 Персональный туман — каждый игрок видит только то, что ему открыл мастер
- 🎨 Тёмный фэнтези-дизайн с золотом, пурпуром и рунами

## 🚀 Быстрый старт

### Локально

```bash
npm install
npm start
```

Открой `http://localhost:3000` в браузере. Каждый игрок — новая вкладка / устройство.

### Деплой на Render (бесплатно)

1. Залей репозиторий на GitHub.
2. Зайди на [render.com](https://render.com), нажми **New +** → **Blueprint**.
3. Подключи свой репозиторий — Render сам подхватит `render.yaml` и развернёт сервис.

Альтернативно вручную:
- **New Web Service** → выбери репозиторий
- Runtime: **Node**
- Build Command: `npm install`
- Start Command: `npm start`
- Health Check Path: `/healthz`
- Instance Type: **Free**

Через ~1 минуту получишь публичную ссылку вида `https://vrata-sudby.onrender.com` — раздавай друзьям.

## 🎮 Как играть

1. Мастер открывает ссылку, жмёт **Мастер** и вводит пароль `Ничтожество`.
2. Игроки открывают ту же ссылку, жмут **Игрок**, вводят имя и цвет.
3. Мастер загружает карту, настраивает сетку, расставляет NPC.
4. В панели тумана (справа сверху) мастер выбирает игрока и кистью открывает ему видимые области.
5. Игроки таскают свои токены, кидают кубы в чате, шепчутся с мастером.

### Управление доской

- **Колесо мыши** — зум
- **Пробел + перетаскивание** или **средняя кнопка** — панорамирование
- **ПКМ по токену** (мастер) — меню редактирования
- **`/r 2d20+3`** — бросок кубов в чате
- **`/w текст`** — приватное сообщение мастеру

## 🛠 Стек

- Backend: **Node.js + Express + ws** (WebSocket)
- Frontend: чистые HTML/CSS/JS (никаких фреймворков)
- Состояние синхронизируется через сервер, хранится в памяти + в `localStorage` клиента

## ⚠ Ограничения free-плана Render

Бесплатный инстанс засыпает после 15 минут неактивности и просыпается ~30 сек при первом запросе. Состояние игры хранится в оперативной памяти сервера — если сервер перезапустится, данные потеряются (но клиенты, оставшиеся онлайн, помогут восстановить актуальное состояние). Для длительных кампаний рекомендую платный план или добавить БД.
